# 工作记忆与历史记忆管理设计

## 背景

工作记忆与历史记忆管理模块负责沉淀 agent 在运行过程中的原始执行轨迹，是长期记忆学习、问题推荐、行为追踪和审计分析的上游数据源。

该模块当前以三层核心对象组织运行记录：

- `conversation`：会话级容器，承载用户多轮连续交互
- `chat`：单轮问答，承载一次用户提问和模型最终回答
- `action`：中间动作，承载模型在一次问答中的执行步骤

## 目标

- 为 agent 运行过程提供稳定的结构化记录模型
- 支持按会话、对话、动作三层维度追踪执行链路
- 为长期记忆学习和问题推荐提供可消费的上游数据
- 为后续扩展工具调用、知识检索、外部接口调用等动作类型预留空间

## 非目标

- 不负责长期知识抽取与归纳
- 不负责向量化和语义召回
- 不负责推荐排序策略本身

## 模块边界

### 上游

- 用户请求入口
- agent 执行引擎
- 工具调用链路
- 知识检索与外部接口调用逻辑

### 下游

- 学习服务
- 推荐服务
- 审计与回放能力
- 运营分析和问题排查能力

## 核心数据模型

### conversation

| 字段 | 说明 |
| --- | --- |
| `conversationId` | 会话唯一标识 |
| `userId` | 用户标识 |
| `sessionKey` | 会话关联键，可选 |
| `status` | 会话状态，如进行中、已结束 |
| `startedAt` | 会话开始时间 |
| `endedAt` | 会话结束时间，可选 |
| `metadata` | 会话级扩展信息 |

约束建议：

- 同一 `conversationId` 必须唯一
- 会话内 `chat` 必须按时间或序号可排序

### chat

| 字段 | 说明 |
| --- | --- |
| `chatId` | 对话唯一标识 |
| `conversationId` | 所属会话标识 |
| `userMessage` | 用户输入 |
| `finalAnswer` | 模型最终输出 |
| `modelName` | 参与回答的模型标识 |
| `sequenceNo` | 在会话中的顺序号 |
| `createdAt` | 对话创建时间 |
| `contextRef` | 上下文引用信息 |

约束建议：

- `chat` 必须归属于一个 `conversation`
- `sequenceNo` 在单个会话内应唯一且单调递增

### action

| 字段 | 说明 |
| --- | --- |
| `actionId` | 动作唯一标识 |
| `chatId` | 所属对话标识 |
| `actionType` | 动作类型，如工具调用、接口调用、知识查询 |
| `sequenceNo` | 在对话中的执行顺序 |
| `input` | 动作输入 |
| `output` | 动作输出 |
| `status` | 动作执行状态 |
| `startedAt` | 动作开始时间 |
| `finishedAt` | 动作结束时间 |
| `metadata` | 扩展属性 |

约束建议：

- `action` 必须归属于一个 `chat`
- 同一 `chat` 下的 `action` 需要保留真实执行顺序

## 关系模型

```text
conversation 1 --- n chat 1 --- n action
```

关系原则：

- `conversation` 是聚合根视角下的最上层容器
- `chat` 是用户可感知的单轮交互单元
- `action` 是面向系统追踪与解释的内部执行明细

## 接口边界

当前建议按能力拆分接口，而不是强绑定到具体技术实现。

### 会话管理接口

职责：

- 创建会话
- 查询会话详情
- 查询会话下的对话列表
- 结束或关闭会话

### 对话记录接口

职责：

- 记录一次用户提问
- 记录模型最终回答
- 查询单轮对话详情

### 动作记录接口

职责：

- 追加中间动作
- 查询对话下动作列表
- 支持按动作类型过滤

## 一致性与写入策略

- `conversation`、`chat`、`action` 建议具备明确的写入顺序约束
- 至少应保证 `chat` 写入时所属 `conversation` 已存在
- 至少应保证 `action` 写入时所属 `chat` 已存在
- 对于流式回答或分阶段动作，可考虑先写占位记录，再补全最终结果

## 查询与索引建议

- 按 `conversationId` 查询完整链路
- 按 `chatId` 查询动作序列
- 按 `userId + 时间范围` 查询历史会话
- 按 `actionType` 查询动作分布

## 演进方向

- 支持更多 `actionType`
- 支持流式输出过程中的阶段记录
- 支持对 `action` 增加耗时、错误码、重试信息
- 支持对话回放与可视化链路查看

## 风险与注意事项

- 如果 `action` 记录粒度不统一，后续学习与分析成本会变高
- 如果 `sequenceNo` 和时间戳定义不清晰，执行链路可能难以可靠重建
- 如果 `metadata` 无约束膨胀，容易导致模型不稳定和治理困难
