# 长期记忆管理设计

## 背景

长期记忆管理模块负责承接学习服务从历史记忆中提炼出的稳定知识，并提供按类型、按实例库、按语义相关性进行管理和检索的能力。

该模块是“从运行记录走向可复用知识”的核心承载层，当前明确依赖 OpenSearch 作为存储与检索基础，并支持向量检索。

## 目标

- 支持多种长期记忆类型的统一管理
- 支持每种类型下的多实例库隔离
- 支持记忆内容的结构化存储与向量检索
- 为推荐、召回和推理提供稳定的知识读取能力

## 非目标

- 不负责学习算法本身
- 不负责生成 embedding 的具体模型实现
- 不负责推荐结果的最终排序

## 模块边界

### 上游

- 学习服务
- 记忆抽取与知识归纳流程

### 下游

- 推荐服务
- 检索召回服务
- 推理增强链路

## 记忆类型模型

当前至少支持以下类型：

- 用户偏好记忆
- 事实记忆
- 概念记忆
- 程序性记忆

建议使用统一的类型枚举，并避免在调用方侧散落类型常量。

## 实例库模型

实例库用于隔离不同学习任务、租户、业务场景或知识域。

### 建议字段

| 字段 | 说明 |
| --- | --- |
| `memoryStoreId` | 实例库唯一标识 |
| `memoryType` | 所属记忆类型 |
| `name` | 实例库名称 |
| `scope` | 适用范围，如用户级、任务级、领域级 |
| `status` | 状态 |
| `metadata` | 扩展配置 |

## 长期记忆对象模型

### 建议字段

| 字段 | 说明 |
| --- | --- |
| `memoryId` | 记忆唯一标识 |
| `memoryType` | 记忆类型 |
| `memoryStoreId` | 所属实例库 |
| `content` | 记忆正文 |
| `summary` | 可选摘要 |
| `sourceRef` | 来源引用，如 conversation/chat/action |
| `embedding` | 向量表示 |
| `tags` | 标签 |
| `score` | 可选质量评分 |
| `createdAt` | 创建时间 |
| `updatedAt` | 更新时间 |
| `metadata` | 结构化扩展信息 |

## 存储设计

### OpenSearch 角色

- 作为长期记忆主存储之一
- 承载结构化过滤
- 承载向量检索

### 索引设计建议

至少考虑以下维度：

- `memoryType`
- `memoryStoreId`
- `content`
- `tags`
- `sourceRef`
- `createdAt`
- `embedding`

### 分片与隔离建议

- 可以按环境、租户或业务域分索引
- 也可以在统一索引中通过 `memoryType + memoryStoreId` 做逻辑隔离
- 需要在成本、检索性能和治理复杂度之间做平衡

## 检索模型

### 精确过滤

适用场景：

- 按记忆类型查询
- 按实例库查询
- 按标签或来源查询

### 向量检索

适用场景：

- 语义相似召回
- 与当前问题或上下文最相关记忆的查找

### 混合检索

建议支持：

- 先过滤后向量检索
- 向量召回后再按业务规则重排

## 接口边界

### 记忆写入接口

职责：

- 接收学习服务产出的记忆对象
- 校验类型、实例库和必要元数据
- 写入 OpenSearch

### 记忆查询接口

职责：

- 按类型、实例库、标签、来源等条件查询
- 返回结构化记忆结果

### 语义检索接口

职责：

- 接收查询向量或查询文本
- 执行向量检索
- 返回相关记忆及分数

## 一致性与更新策略

- 写入记忆时需要明确幂等键或去重策略
- 同一记忆的更新应保证可追踪
- 实例库删除或停用时需要定义记忆数据保留策略

## 演进方向

- 支持更多记忆类型
- 支持多级质量评分和反馈回写
- 支持更细粒度的生命周期管理
- 支持跨实例库联合召回

## 风险与注意事项

- 向量检索效果高度依赖 embedding 质量
- 实例库划分过细会带来治理和检索成本
- 若来源引用不完整，长期记忆会失去可解释性
