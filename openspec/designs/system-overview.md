# 系统总体设计

## 背景

本文档用于沉淀当前记忆系统的长期结构认知，帮助团队在 OpenSpec 工作流中快速理解系统边界、核心能力和关键数据流。

当前系统围绕 agent 运行过程中的记忆管理与问题联想能力展开，整体可以分为三部分：

- 工作记忆与历史记忆管理
- 长期记忆管理
- 问题推荐与问题联想

## 模块视图

| 模块 | 职责 | 上游 | 下游 |
| --- | --- | --- | --- |
| 工作记忆与历史记忆管理 | 记录 agent 运行期间的会话、对话和动作轨迹 | 用户请求、agent 执行过程 | 学习服务、推荐服务、审计查询 |
| 长期记忆管理 | 存储学习服务产出的稳定知识并提供检索能力 | 学习服务 | 推荐服务、推理服务、检索调用方 |
| 问题推荐与问题联想 | 基于历史记忆、问题库和实体生成候选问题 | 当前上下文、历史记忆、长期记忆、预置问题库 | 前端调用方、对话引导流程 |

## 核心领域对象

### 工作记忆与历史记忆

| 对象 | 说明 |
| --- | --- |
| `conversation` | 会话信息，由用户多次问答组成一个会话 |
| `chat` | 单次对话信息，由用户一次提问和模型最终返回组成 |
| `action` | 对话过程中的中间动作，例如工具调用、接口调用、知识查询等 |

它们的关系为：

- 一个 `conversation` 下包含多条 `chat`
- 一条 `chat` 下包含多条 `action`
- `action` 按执行顺序记录模型从提问到最终回答之间的关键过程

### 长期记忆

长期记忆由学习服务根据历史记忆学习得到，当前至少包含以下类型：

- 用户偏好记忆
- 事实记忆
- 概念记忆
- 程序性记忆

每种类型下可以定义多个实例库，用于隔离不同学习任务或业务场景产出的记忆集合。

### 问题推荐

问题推荐当前包含多种生成路径：

- 基于历史记忆推荐
- 基于预置典型问题库提取
- 基于历史记忆中的实体提取进行推荐

不同推荐路径可能有不同的接口和数据模型，但都应收敛到统一的能力边界和兼容性约束之下。

## 运行时边界

- 应用入口：记忆系统服务接口、推荐接口及相关内部调用入口
- 数据库：工作记忆与历史记忆的结构化存储待按真实实现补充
- 外部依赖：学习服务、OpenSearch、可能存在的实体抽取或知识服务
- 检索能力：长期记忆使用 OpenSearch 存储并支持向量检索
- 异步链路：学习服务根据历史记忆异步学习并回写长期记忆

## 横切关注点

- 数据追踪：需要能够按会话、对话、动作完整追踪 agent 执行链路
- 兼容性：推荐接口与长期记忆模型演进时需要控制调用方兼容性
- 检索质量：长期记忆的向量检索效果直接影响推荐与召回质量
- 可扩展性：长期记忆类型、实例库和推荐策略都需要支持持续扩展
- 审计与治理：历史动作、记忆写入和推荐来源需要具备可解释性

## 相关 OpenSpec 能力

当前系统建议至少维护以下长期 spec：

- `working-memory-management`
- `long-term-memory-management`
- `question-recommendation`

后续如果某一块能力持续扩大，可以继续细分为更小的 capability，例如：

- `memory-learning-pipeline`
- `entity-extraction-recommendation`
- `memory-retrieval-ranking`
