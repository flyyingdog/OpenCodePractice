# 用户认证能力

## Purpose

用户认证能力用于识别访问系统的用户身份，并建立可被后续授权流程使用的登录态。

## Requirement: 用户可通过受支持的身份凭据发起登录

系统必须允许用户使用受支持的身份凭据发起登录，并在凭据有效时建立登录态。

### Scenario: 有效凭据登录成功

- GIVEN 用户提供合法凭据
- WHEN 用户发起登录请求
- THEN 系统返回成功结果
- AND 系统建立与该用户对应的登录态

### Scenario: 无效凭据登录失败

- GIVEN 用户提供非法或不匹配的凭据
- WHEN 用户发起登录请求
- THEN 系统拒绝登录
- AND 系统返回明确的失败结果

## Requirement: 未认证请求不能访问需要登录的受保护能力

系统必须拒绝未建立有效登录态的调用方访问需要认证的受保护能力。

### Scenario: 未登录访问受保护接口被拒绝

- GIVEN 调用方未建立有效登录态
- WHEN 调用方访问受保护能力
- THEN 系统拒绝该请求
- AND 系统返回认证失败结果

## Requirement: 登录态失效后需要重新认证

系统必须在登录态失效、过期或被显式撤销后要求用户重新认证。

### Scenario: 失效登录态不能继续访问受保护能力

- GIVEN 调用方持有已失效的登录态
- WHEN 调用方访问受保护能力
- THEN 系统拒绝该请求
- AND 系统要求重新认证
