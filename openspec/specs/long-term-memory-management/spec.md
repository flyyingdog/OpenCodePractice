# 长期记忆管理

## Purpose

长期记忆管理能力用于承载由学习服务基于历史记忆学习出的稳定知识，并为后续检索、召回和推理提供结构化、可扩展的长期知识存储。

## Requirements

### Requirement: 系统必须支持多种长期记忆类型

系统 MUST 支持用户偏好、事实记忆、概念记忆和程序性记忆等长期记忆类型，并允许不同类型采用统一的管理方式。

#### Scenario: 学习服务写入不同类型的长期记忆

- **GIVEN** 学习服务生成了不同语义类型的记忆结果
- **WHEN** 系统接收长期记忆写入请求
- **THEN** 系统能够识别记忆类型
- **AND** 将记忆按对应类型归档保存

#### Scenario: 调用方按记忆类型检索

- **GIVEN** 系统中已存在多种类型的长期记忆
- **WHEN** 调用方指定某一类型进行查询
- **THEN** 系统只返回目标类型下的记忆结果
- **AND** 不混入其它类型的记忆数据

### Requirement: 系统必须允许每种记忆类型下管理多个实例库

系统 MUST 允许每种长期记忆类型下定义多个实例库，以便隔离不同学习任务、场景或业务域产生的记忆。

#### Scenario: 同一记忆类型下存在多个实例库

- **GIVEN** 某种记忆类型下存在多个实例库
- **WHEN** 学习服务写入新的记忆内容
- **THEN** 系统将记忆写入指定实例库
- **AND** 该记忆不会自动混入其它实例库

#### Scenario: 按实例库范围查询记忆

- **GIVEN** 某种记忆类型下存在多个实例库
- **WHEN** 调用方指定实例库进行查询
- **THEN** 系统仅在目标实例库范围内执行检索
- **AND** 返回结果带有实例库归属信息

### Requirement: 系统必须支持基于 OpenSearch 的长期记忆持久化

系统 MUST 使用 OpenSearch 持久化长期记忆，并保证记忆内容及其检索所需元数据可被可靠存储和读取。

#### Scenario: 长期记忆写入 OpenSearch

- **GIVEN** 学习服务已产出可持久化的长期记忆
- **WHEN** 系统执行存储流程
- **THEN** 记忆内容被写入 OpenSearch
- **AND** 与记忆类型、实例库和检索相关的元数据一并保存

#### Scenario: 持久化后的记忆可被再次读取

- **GIVEN** 某条长期记忆已成功存储
- **WHEN** 调用方按标识或检索条件读取该记忆
- **THEN** 系统能够从 OpenSearch 返回对应记忆
- **AND** 返回内容与写入时的核心信息一致

### Requirement: 系统必须支持向量检索长期记忆

系统 MUST 支持对长期记忆执行向量检索，以便根据语义相似度召回相关记忆。

#### Scenario: 使用语义相似查询召回记忆

- **GIVEN** 系统中已存在可进行向量检索的长期记忆
- **WHEN** 调用方提交一个语义查询
- **THEN** 系统基于向量检索返回相关记忆
- **AND** 返回结果按相关性排序

#### Scenario: 向量检索结合类型或实例库过滤

- **GIVEN** 系统中存在跨类型、跨实例库的长期记忆
- **WHEN** 调用方同时指定语义查询和过滤条件
- **THEN** 系统在过滤范围内执行向量检索
- **AND** 返回符合过滤条件的相关结果
